# Llama3 最快速度学习路线

> 目标：用最少的时间，从底层原理到完整模型，彻底掌握 Llama3 的核心实现。
> 建议顺序：**先拆组件 → 再学加速 → 最后拼成完整模型**，避免一上来就被海量代码淹没。

---

## 阶段一：核心组件拆解（GQA + RoPE）
**预计时间**：2–3 小时  
**核心目标**：亲手写出 Llama3 注意力机制的两个基石，理解张量变换的每一步。

Llama3 的 Attention 不是标准多头注意力，而是 **GQA（分组查询注意力）** 配合 **RoPE（旋转位置编码）**。这两个模块相对独立，代码量小，最适合作为突破口。

- **对应文件**：`手撕GQA & RoPE.ipynb`
- **必会要点**：
  - GQA 如何通过共享 Key/Value 头来减少显存与计算
  - RoPE 的复数旋转、角度频率 `θ` 的物理意义
  - 从数学公式到 PyTorch 张量操作的映射

> 💡 **技巧**：不要只看，要跟着 notebook 逐行运行，打印张量 shape，验证每一步是否符合预期。

---

## 阶段二：推理加速 —— KV Cache
**预计时间**：2–3 小时  
**核心目标**：理解 Transformer Decoder 在自回归生成时的性能瓶颈，并亲手实现 KV Cache 优化。

在阶段一你只关注了“单步” Attention 的计算。但实际推理是逐个 token 生成的，如果不缓存历史 Key/Value，每次都要重复计算，复杂度是 `O(n²)`。KV Cache 将其降到 `O(n)`。

- **对应文件**：`带kv cache的Transformer.ipynb`
- **必会要点**：
  - 带 KV Cache 与不带 KV Cache 的 `forward` 差异
  - `past_key_values` 的拼接与更新逻辑
  - 为什么「用空间换时间」在长序列生成中能带来 2–10 倍加速
  - 自己动手对比两种实现的推理耗时

> 💡 **技巧**：把这里写的 Decoder 框架记下来，阶段三的 Llama3 就是在这个骨架上填更复杂的 Attention 和前馈网络。

---

## 阶段三：端到端实战 —— 从零手撕 Llama3
**预计时间**：1–2 天（可分多次完成）  
**核心目标**：把前面学的所有组件串起来，用原生 PyTorch 完成一次真实权重的推理。

这是信息量最大的 notebook，但因为你已经在阶段一、二打好了地基，现在只需要「按图索骥」：知道每一行代码对应的是哪个模块，不再陌生。

- **对应文件**：`手撕llama3/llama3-from-scratch-Copy1.ipynb`
- **必会要点（按 notebook 章节）**：
  1. **Tokenizer**：BPE 分词逻辑，如何把文本变成 token id
  2. **读取真实权重**：从 Meta 原版 `consolidated.00.pth` 逐张量加载
  3. **Embedding**：`[seq_len] → [seq_len × dim]`
  4. **RMSNorm**：与 LayerNorm 的区别，`norm_eps` 的作用
  5. **Attention 完整链路**（这里会用到阶段一的 GQA + RoPE）：
     - Q/K/V 投影 → 多头拆分 → RoPE 旋转 → Mask → Softmax → Attention 输出 → Wo 投影
  6. **残差连接 + FFN（SwiGLU）**：Llama3 的前馈网络结构
  7. **32 层循环**：理解每一层都在「提炼」嵌入信息
  8. **输出解码**：用最后一个 token 的 embedding 预测下一个词

> 💡 **技巧**：notebook 里配了大量示意图（`images/` 目录），遇到 shape 转换看不明白时，直接对照图片。
